# KX Development Agent Workflow

모든 KX 개발 에이전트는 구현 전에 아래 순서를 따른다.

## 1. READ

- `spec.md`, `plan.md`, 관련 대상 파일을 먼저 읽는다.
- 기존 코드 패턴과 제약을 파악한다.

## 2. EXTRACT

- 구현해야 할 요구사항을 체크리스트로 추출한다.
- 파일, 인터페이스, 유효성 검증, 에러 처리, 테스트 요구를 누락 없이 적는다.

## 3. CODE

- 체크리스트를 기준으로 구현한다.
- 기존 패턴을 유지하고, 필요한 범위만 수정한다.

## 4. VERIFY

- 문서와 구현을 다시 대조한다.
- 관련 테스트, 타입체크, 린트를 가능한 가장 좁은 범위로 실행한다.
- 실패 시 원인을 수정하고 재검증한다.
